binarySearch()

Cada clique no botão binarySearch() representa uma iteração do algoritmo, mostrando atualizações pra cada um: mid (meio), guess (chute), e o resultado da iteração.

Target: -
Low: -
High: -
Mid: -
Guess: -
Result: -

Esse projeto tem como objetivo explicar de forma visual a execução dos principais algoritmos ensinados no livro grokking algorithms. Começando pelo algoritmo binarySearch(), que funciona dividindo repetidamente o intervalo de busca pela metade:
1. Compara o elemento do meio com o valor buscado
2. Se mid = target, encontramos o valor buscado
3. Se mid > target, ou seja, se o valor do meio é maior que o valor buscado, o ultimo elemento do array (high) passa a ser o elemento anterior ao meio (mid-1)
4. Se mid < target, ou seja, se o valor do meio é menor que o valor buscado, o primeiro elemento do array (low) passa a ser o elemento posterior ao meio (mid+1)
